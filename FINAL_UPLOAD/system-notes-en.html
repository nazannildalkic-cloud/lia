System Notes

This page describes how Nanil is conceived. No promises. No roadmap. Just system logic.

1. App-first architecture

The app is the system.

Hardware is interchangeable. Logic is not.

2. Sensors as input

Sensors provide raw data. Meaning arises only through context and time.

3. Patterns instead of KPIs

Nanil optimises for relationships, not daily values.

4. Time as structure

Progression, repetition and deviation are more central than snapshots.

5. Early-stage principle

Architecture before features. Understanding before scaling.
